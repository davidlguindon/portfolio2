Problem

Black Desert Online has an economic system that has very low profit margins on elastic goods, this leads to it being difficult to know what to produce as a new user. To compound this problem, when someone does find a profitable item to produce, they do not share it, because the market can not handle many users producing the same good.

Solution

Desert Paradigm allows users to imput their personal data into a spreadsheet and find the most profitable item for them to create given the current market. This can be done even though their is no public API available This is because users are able to audit out of date items and update their information, then share it with other users.

Discovery and Research

1111111111The most difficult part of this task was creating a form that would easily allow cross referencing items and creation methods. There are hundreds of variables to consider when trying to figure out the best item to create, and we had to figure out a way to organize them in a way that would make sense to the user, and programmers alike. We also had to try to minimize the amount of user input by auto filling in as many variables as possible. Another consideration was that not all users were interested in the same data, so we had to find a way to show as much important data to the user as possible on a single to the user as possible, while keeping the need for them to customize to a minimum.

Reflection

This project was an exercise for me to learn and solidify my design process, it was created for myself and I was not planning on using it for a portfolio piece, so the audience of many of my documents are not what I would have liked. While designing it I made it in a way that I could complete the whole project on my own and would like to finish and publish the site in my spare time. My two largest regrets on the project as it stands currently are that I did not design the site to have full functionality on mobile, roughly half of the features are cut out, and if I was going to do it again I would design mobile first. The reason I did not is because the game the site is intended for high end gaming computers. Not having the full functionality on mobile still bothers me because with better planning I could have incorporated it easily. My second regret is that I have yet to find a way to show item data in a way that all my testers easily understood. Each iteration was getting better, but I decided to end the project before everything was ironed out to my satisfaction. In my last trial I did not even take the time to test the contrast ratio of my new color coded design. Overall I had a very fun time building this project, and would appreciate it if someone who is looking at it keeps in mind that I was focused on speed and learning the design process more than fine tuning and making a project I would actually send to a client.
